Apache Maven的安装和使用

发表于 2017-07-08

安装

  • 官网下载安装包，解压。
  • 配置JDK和M2_HOME。

创建maven项目

  • 创建
    mvn archetype:generate -DgroupId=com.codoter.example -DartifactId=example -Dpackage=com.codoter.example -Dversion=1.0-SNAPSHOT

  • 打包（选择默认的maven-archetype-quickstart模型）
    mvn package

  • 运行
    java -cp target/example-1.0-SNAPSHOT.jar com.codoter.example.App

POM (Project Object Model)

  • 项目所有的配置都放置在pom文件中。定义项目类型、名字，依赖管理，定制插件的行为等。

Maven插件

  • 使用mvn archetype:generate 命令来生成一个项目时，archetype是插件的名字，generate是目标。意思是maven执行archetype插件的generate目标。插件通常写成：pluginId:goalId。
  • 一个目标是一个工作单元，插件是一个或多个目标的集合。如Jar插件包含建立Jar文件的目标，Compiler插件包含编译源代码和单元测试代码的目标。Surefire插件则是运行单元测试。

Maven生命周期

  • 打包时执行的命令：mvn package。这里的package指的是一个maven生命周期阶段（life phrase）。生命周期指的是项目的构件过程，包含了一些列的有序阶段。
  • 一个生命周期可以绑定多个插件目标，maven在构建过程中会执行每个阶段的所有插件目标。

Maven依赖管理

  • 在dependecies中定义依赖关系，用maven坐标来确定一个依赖。
  • scope决定依赖关系适用范围。例如对于junit的scope为test，那么只有在执行compiler:testCompile and surefire:test目标的时候才会被加入到classpath中执行。执行compiler:compile目标是拿不到junit的。
  • 当scope被指定为provided时，意思是JDK或者容器会提供所需的jar文件。比如web开发时，编译需要servlet API jar文件，但是打包的时候不需要把这个jar打在war中，因为servlet容器或者应用服务器会提供。
  • scope默认值是compile，即在任何时候都会被包含在classpath中（包括打包）。

Maven库

  • 首次使用maven命令的时候会从maven默认的远程库（http://repo1.maven.org/maven2）下载一些核心插件和jar文件。
  • 当默认远程库没有需要的jar文件时，有两种选择：自己手动下载和安装jar到本地库或者内部设置定制库。
  • 本地库(~/.m2/repository)。当maven查找需要的jar时，会先从本地库查找，找不到时再去远程库查找。
  • mvn install可以把自己创建的项目安装到本地库，这样其他项目就可以通过maven坐标来建立依赖关系了。
